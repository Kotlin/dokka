# Gradle v2 examples

These projects demonstrate how to use Dokka Gradle Plugin in v2 mode.

| Project                                                          | Description                                                                                                                                                            |
|------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [basic-gradle-example](basic-gradle-example)                     | A basic Dokka example project.                                                                                                                                         |
| [composite-build-example](composite-build-example)               | How to use Dokka to generate documentation for a [composite build](https://docs.gradle.org/current/userguide/composite_builds.html).                                   |
| [custom-format-example](custom-format-example)                   | Demonstrates how to override `.css` styles and add custom images as assets, allowing you to change the logo used in the header.                                        |
| [java-example](java-example)                                     | Dokka can be used to document Java code too.                                                                                                                           |
| [javadoc-example](javadoc-example)                               | Use the Dokka Javadoc plugin to create a HTML site that emulates Javadoc.                                                                                              |
| [kotlin-as-java-example](kotlin-as-java-example)                 | Demonstrates how to apply the Dokka plugin, [Kotlin as Java](../../dokka-subprojects/plugin-kotlin-as-java).                                                           |
| [kotlin-multiplatform-example](kotlin-multiplatform-example)     | Produce Dokka for a simple [Kotlin Multiplatform](https://kotlinlang.org/docs/multiplatform.html) project.                                                             |
| [library-publishing-example](library-publishing-example)         | How to integrate Dokka into the publishing process of a library, adding documentation generated by Dokka as artifacts.                                                 |
| [multimodule-example](multimodule-example)                       | how to apply and configure Dokka in a [multi-project build](https://docs.gradle.org/current/userguide/multi_project_builds.html).                                      |
| [versioning-multimodule-example](versioning-multimodule-example) | Configuration of Dokka's [versioning plugin](../../dokka-subprojects/plugin-versioning), which allows readers to navigate through different versions of documentation. |
